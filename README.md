# Фабрика навыков

Ты поработал с Claude — и хочешь, чтобы дальше он делал это сам, без объяснений с нуля каждый раз.

Навык (скилл) — это файл-памятка, который Claude в следующий раз подхватывает сам. Обычно такой файл пишут руками. **Здесь его пишет Claude, а ты только разговариваешь.**

Ты не автор навыка. Ты источник. Приносишь материал — фабрика собирает.

---

## Пять дверей: из чего можно собрать навык

Материал входит через одну из пяти дверей. Через одну ты, скорее всего, уже проходил — просто не знал, что это она.

| Дверь | Что приносишь | Что говоришь |
|-------|---------------|--------------|
| **① Дело** | работу, которую вы только что сделали в чате | «закрепи это» |
| **② Видео** | запись экрана, где ты руками гоняешь рутину | «посмотри, что я делаю, собери навык» |
| **③ Образец** | готовую вещь или чужой навык, который хочешь повторять | «научись делать так же» |
| **④ Знание** | книгу, конспект, методичку | «собери навык, который это применяет» |
| **⑤ Ты сам** | ничего — только опыт в голове | «распакуй меня, задай вопросы и собери навык» |

Фабрика сама поймёт, какая это дверь, и поведёт за руку: достанет процедуру, напишет навык, проверит, что он реально срабатывает, и отдаст тебе простыми словами — где лежит и как звать. Ни YAML, ни терминала.

## Установка

### Способ 1 — командой (терминальный Claude Code)

Открой Claude Code и вставь две строки одну за другой:

```
/plugin marketplace add rusanovproject-dotcom/fabrika-navykov
/plugin install fabrika-navykov@fabrika-navykov
```

Готово. Проверить, что встало — набери `/skills`, фабрика будет в списке.

### Способ 2 — файлами (если `/plugin` не сработал)

Если в ответ на команду ты видишь **«/plugin isn't available in this environment»** — значит твой клиент Claude Code не поддерживает плагины (так бывает в веб- и десктоп-версии). Ставим напрямую файлами, Claude всё сделает сам:

**Шаг 1.** На этой странице GitHub нажми зелёную кнопку **Code → Download ZIP**. Скачается архив — распакуй двойным щелчком. Получится папка `fabrika-navykov` (или `fabrika-navykov-main`).

**Шаг 2.** Открой Claude Code и вставь это одним сообщением — ничего не меняя:

```
Найди у меня на компьютере скачанную папку fabrika-navykov (может называться
fabrika-navykov-main, скорее всего в Загрузках). Внутри неё есть папка
skills/fabrika-navykov — установи её как скилл: скопируй содержимое
skills/fabrika-navykov в ~/.claude/skills/fabrika-navykov (имя папки — ровно
fabrika-navykov, внутри должен оказаться SKILL.md). Потом проверь через /skills
и скажи, что получилось.
```

Claude найдёт папку, положит куда надо и отчитается. Терминал открывать не нужно — это его работа.

**Важно:** ставить именно так, а **не** через `npx skills add` — та команда кладёт файлы в другую папку, Claude их не увидит, и будет тишина без ошибки.

## Как пользоваться

Просто скажи своими словами, что хочешь закрепить. Примеры:

- «Закрепи это — чтобы в следующий раз само»
- «У меня конспект по переговорам, собери навык, который это применяет»
- «Я пятнадцать лет это делаю на автомате — распакуй меня и собери навык»
- «Научись делать такие же страницы, вот образец»

Фабрика задаст пару вопросов живым языком, соберёт навык и покажет тебе файл до того, как сохранит.

## Что внутри

- `skills/fabrika-navykov/SKILL.md` — сама фабрика: тракт из 11 шагов от материала до готового навыка.
- `references/vhody.md` — пять входов подробно: как добыть материал из каждого.
- `references/arhetipy.md` — девять форм навыка (процедура, оценщик, генератор, страж…).
- `references/opisaniya.md` — формула описания-вывески, которая решает, сработает навык или нет.
- `evals/` — проверки самой фабрики и фразы, на которых она должна подниматься.

Фундамент методологии — официальный `skill-creator` от Anthropic, адаптированный для работы голосом, без написания файлов руками.
